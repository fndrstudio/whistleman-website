<task type="auto">
  <name>Task 1: Add shimmer animation to primary CTAs</name>
  <files>assets/css/main.css</files>
  <action>
Add shimmer/shine animation to primary CTA buttons. Insert after the existing button hover/micro-interaction styles (around line 3700 or after the button styles section).

**1. Add shimmer keyframes animation:**
```css
/* Shimmer animation for primary CTAs (Phase 7) */
@keyframes shimmer-sweep {
  0% { transform: translateX(-100%); }
  20%, 100% { transform: translateX(100%); }
}
```

**2. Add shimmer effect to primary CTA buttons:**
```css
/* Apply shimmer to primary CTAs */
.hero .btn-get-started,
.php-email-form button[type="submit"],
.btn-footer {
  position: relative;
  overflow: hidden;
}

.hero .btn-get-started::after,
.php-email-form button[type="submit"]::after,
.btn-footer::after {
  content: '';
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background: linear-gradient(
    120deg,
    transparent 40%,
    rgba(255, 255, 255, 0.25) 50%,
    transparent 60%
  );
  transform: translateX(-100%);
  animation: shimmer-sweep 4s ease-in-out infinite;
  pointer-events: none;
}

/* Pause shimmer on hover (button is being interacted with) */
.hero .btn-get-started:hover::after,
.php-email-form button[type="submit"]:hover::after,
.btn-footer:hover::after {
  animation-play-state: paused;
}

/* Reduced motion: disable shimmer */
@media (prefers-reduced-motion: reduce) {
  .hero .btn-get-started::after,
  .php-email-form button[type="submit"]::after,
  .btn-footer::after {
    animation: none;
    display: none;
  }
}
```

**Important:** Use `transform: translateX()` NOT `left` property for animation. Transform is GPU-accelerated and won't cause jank.

The shimmer timing (4s with 20% active sweep, 80% rest) provides attention-grabbing effect without being distracting per research recommendations.

<task type="auto">
  <name>Task 3: Fix mobile contact form single-column layout</name>
  <files>assets/css/main.css</files>
  <action>
Fix the mobile contact form layout to use single-column stacking. Add to the mobile media query section (around @media max-width: 767px).

**Add mobile form responsive styles:**
```css
/* Mobile contact form - single column layout (Phase 7) */
@media (max-width: 767px) {
  #hero-contact .php-email-form .row {
    flex-direction: column;
  }

  #hero-contact .php-email-form [class*="col-"] {
    width: 100%;
    max-width: 100%;
    flex: 0 0 100%;
    padding-left: 0;
    padding-right: 0;
  }

  #hero-contact .php-email-form {
    padding: 16px;
  }

  #hero-contact .php-email-form input,
  #hero-contact .php-email-form textarea {
    font-size: 16px; /* Prevents iOS zoom on focus */
    padding: 14px 16px;
  }

  #hero-contact .contact-subtitles {
    font-size: 16px;
    margin-bottom: 8px;
  }

  /* Better spacing between fields */
  #hero-contact .php-email-form .gy-4 > * {
    margin-bottom: 16px;
  }

  /* Full-width submit button */
  #hero-contact .php-email-form button[type="submit"] {
    width: 100%;
    padding: 16px;
    font-size: 16px;
  }
}
```

**Critical:** The `font-size: 16px` on inputs is essential - iOS Safari automatically zooms when focusing inputs smaller than 16px, which is a poor UX.

If the contact form also exists on the contact.html page (not just hero-contact), apply similar rules:

```css
    max-width: 100%;
    flex: 0 0 100%;
  }

  .contact .php-email-form input,
  .contact .php-email-form textarea {
    font-size: 16px;
  }

  .contact .php-email-form button[type="submit"] {
    width: 100%;
  }
}
```
